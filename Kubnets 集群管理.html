============================================= Master 节点安装 ==============================
一. 安装 etcd 
  1.  yum install etcd
  2. 检查命令   etcd
  3. 配置 etcd-ctl 环境变量  export  ETCDCTL_API=3
  4. 启动etcd 测试是成功   etcdctl put testkey "hello word"      etcdctl get testkey   正常输出说明成功
    启动etcd服务
    systemctl start etcd
    systemctl enable etcd
    systemctl status etcd
    etcdctl ls
  1.     检测服务是否启动成功 etcdctl cluster-health 无错误代表正常
注意：error listing data dir: /etc/etcd/default.etcd   权限问题   chown etcd:etcd -R /var/lib/etcd/default.etcd
          
注意： 1.yum 安装的会自动生服务启动文件 默认在 vi /usr/lib/systemd/system/etcd.service 
            2.配置文件需要修改IP /etc/etcd/etcd.conf
            3.配置2个地址 --listen-client-urls=http://192.168.3.102:2379,http://127.0.0.1:2379
             
二. 安装 GO 环境
  1.     yum install golang （可能找不到安装文件 此时需要重新设置epel 命令为：   rpm -Uvh http://dl.fedoraproject.org/pub/epel/6/x86_64/epel-release-6-8.noarch.rpm）
（epel是社区强烈打造的免费开源发行软件包版本库，系统包含大概有1万多个软件包）

三. 安装 kubenetes
    到 Github Kubenetes 官方下载最新版 https://github.com/kubernetes/kubernetes/releases 
    wget https://github.com/kubernetes/kubernetes/releases/download/v1.13.6/kubernetes.tar.gz
    解压包 进入 kubenetes 目录执行 ./cluster/get-kube-binaries.sh 安装
    注意： 安装时会去国外网站下载 kubernetes-server-linux-amd64.tar.gz  和 kubernetes-client-linux-amd64.tar.gz 两个文件需要翻墙
     下载完毕后执行
        cd server tar xvf kubernetes-server-linux-amd64.tar.gz
        rm -f *_tag *.tar
        chmod 755 *
    将server\bin 下的所有组件 移动到/usr/bin 目录
        mv * /usr/bin
    所有组件 包括
       1.cloud-controller-manager
        2.hyperkube kubeadm
        3.kube-aggregator
        4.kube-apiserver
        5.kube-controller-manager
        6.kubectl
        7.kubefed
        8.kubelet
        9.kube-proxy
        10.kube-scheduler
    安装完成接下开开始配置 所有组件

  2. 创建 api 服务启动文件 vim /usr/lib/systemd/system/kube-apiserver.service

  4.    
  5. 创建 kube-apiserver 配置文件
      
  1.    添加服务
            systemctl enable kube-apiserver.service
            systemctl start kube-apiserver.service

    问题：Unable to create storage backend: config (&{ /registry [http://192.168.3.102:2380] true true 1000 0xc42013b830 <nil> 5m0s 1m0s}), err (dial tcp             192.168.3.102:2380: connect: connection refused)
    检查etcd配置，让其能正常访问
    
五.  配置 scheduler
    
  • 创建配置文件
cat >/etc/kubernetes/kube-scheduler <<EOF
    KUBE_LOGTOSTDERR="--logtostderr=true"
    KUBE_LOG_LEVEL="--v=4"
    KUBE_MASTER="--master=172.16.0.67:8080"
    KUBE_LEADER_ELECT="--leader-elect"
    EOF
    
  • 创建服务器启动文件
cat>/lib/systemd/system/kube-scheduler.service<<EOF
[Unit]
Description=Kubernetes Scheduler
Documentation=https://github.com/kubernetes/kubernetes
[Service]
EnvironmentFile=-/etc/kubernetes/kube-scheduler
ExecStart=/usr/bin/kube-scheduler \
\${KUBE_LOGTOSTDERR} \
\${KUBE_LOG_LEVEL} \
\${KUBE_MASTER} \
\${KUBE_LEADER_ELECT}
Restart=on-failure
[Install]
WantedBy=multi-user.target
EOF
  • 启动服务
systemctl daemon-reload
systemctl enable kube-scheduler
systemctl restart kube-scheduler

============================================= Node 节点安装 ============================== 
一. 配置 kubelet
  • 下载安装kubenets-node软件包
    wget https://dl.k8s.io/v1.14.0/kubernetes-node-linux-amd64.tar.gz       最新版在这里https://kubernetes.io/docs/setup/release/notes/#node-binaries
tar -zxf kubernetes-node-linux-amd64.tar.gz
cd kubernetes/node/bin
mv * /usr/bin/
  • 创建kubeconfig配置文件 (2个)
cat > /etc/kubernetes/kubelet.kubeconfig <<EOF
apiVersion: v1
kind: Configclusters:
- cluster:
server: http://192.168.3.102:8080
name: local
contexts:
- context:
cluster: local
name: local
current-context: local
EOF
注意： server 是master 节点地址， 横杠一定要定格后面空格

cat> /etc/kubernetes/kubelet <<EOF
# 启用日志标准错误
KUBE_LOGTOSTDERR="--logtostderr=true"
# 日志级别
KUBE_LOG_LEVEL="--v=4"
# Kubelet服务IP地址
NODE_ADDRESS="--address=192.168.3.101"
# Kubelet服务端口
NODE_PORT="--port=10250"
# 自定义节点名称
NODE_HOSTNAME="--hostname-override=192.168.3.101"
# kubeconfig路径，指定连接API服务器
KUBELET_KUBECONFIG="--kubeconfig=/etc/kubernetes/kubelet.kubeconfig"
# 允许容器请求特权模式，默认false
KUBE_ALLOW_PRIV="--allow-privileged=false"
# DNS信息
KUBELET_DNS_IP="--cluster-dns=192.168.3.1"
KUBELET_DNS_DOMAIN="--cluster-domain=cluster.local"
# 禁用使用Swap
KUBELET_SWAP="--fail-swap-on=false"
EOF
  • 创建systemd服务文件
cat>/lib/systemd/system/kubelet.service<<EOF
[Unit]
Description=Kubernetes Kubelet
After=docker.service
Requires=docker.service
[Service]
EnvironmentFile=-/etc/kubernetes/kubelet
ExecStart=/usr/bin/kubelet \
\${KUBE_LOGTOSTDERR} \
\${KUBE_LOG_LEVEL} \
\${NODE_ADDRESS} \
\${NODE_PORT} \
\${NODE_HOSTNAME} \
\${KUBELET_KUBECONFIG} \
\${KUBE_ALLOW_PRIV} \
\${KUBELET_DNS_IP} \
\${KUBELET_DNS_DOMAIN} \
\${KUBELET_SWAP}
Restart=on-failure
KillMode=process
[Install]
WantedBy=multi-user.target
EOF

  • 启动服务
systemctl daemon-reload
systemctl enable kubelet
systemctl start kubelet
二. 安装kube-proxy
  • 新建配置文件
cat>/etc/kubernetes/kube-proxy <<EOF
# 启用日志标准错误
KUBE_LOGTOSTDERR="--logtostderr=true"
# 日志级别
KUBE_LOG_LEVEL="--v=4"
# 自定义节点名称
NODE_HOSTNAME="--hostname-override=192.168.3.101"
# API服务地址
KUBE_MASTER="--master=http://192.168.3.102:8080"
EOF

  • 创建systemd服务文件
cat > /lib/systemd/system/kube-proxy.service<<EOF
[Unit]
Description=Kubernetes Proxy
After=network.target
[Service]
EnvironmentFile=-/etc/kubernetes/kube-proxy
ExecStart=/usr/bin/kube-proxy \
\${KUBE_LOGTOSTDERR} \
\${KUBE_LOG_LEVEL} \
\${NODE_HOSTNAME} \
\${KUBE_MASTER}

[Install]
WantedBy=multi-user.target
EOF
  • 启动服务
systemctl daemon-reload
systemctl enable kube-proxy
systemctl restart kube-proxy
